========================================
   PRIORITY INVERSION DEMO
========================================

SCENARIO:
----------
1. Low task acquires resource (holds for 5 sec)
2. High task needs same resource -> gets BLOCKED
3. Medium task preempts Low task (doesn't need resource)
4. High task stays blocked -> PRIORITY INVERSION!

Task Priorities:
  [HIGH]   - Priority 3 (needs resource)
  [MEDIUM] - Priority 2 (CPU-bound, no resource)
  [LOW]    - Priority 1 (holds resource)
